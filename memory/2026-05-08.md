## Requested
Document the Vercel login/protection issue affecting `brownbio.tech` and how it was resolved, using a structured daily log entry.

## Analysis
`brownbio.tech` and the `brown-biotech-platform` deployment were both protected by Vercel SSO/deployment protection, which caused the public domain to return a login/401 response instead of the homepage. I verified that the custom domain was mapped to the `brown-biotech-platform` deployment via `vercel alias ls` and the Vercel API, then identified the project-level `ssoProtection` setting as the blocking control.

## Decisions with rationale
Disabled project-level `ssoProtection` by PATCHing the Vercel project to `null` with the provided Vercel token so the custom domain would serve publicly without requiring Vercel login. I did not write the token into any logs or notes, keeping credentials out of memory while still recording the operational fix. After the change, I validated the result at both the HTTP layer and in the browser to confirm the site was genuinely public rather than only appearing fixed in API state.

## Outcomes
`brownbio.tech` now returns HTTP 200 instead of 401. Browser verification showed the public Brown Biotech homepage loading successfully, confirming the Vercel protection issue was resolved.
