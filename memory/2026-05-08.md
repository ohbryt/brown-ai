I verified that the custom domain was mapped to the `brown-biotech-platform` deployment via `vercel alias ls` and the Vercel API, then identified the project-level `ssoProtection` setting as the blocking control.

## Decisions with rationale

I did not write the token into any logs or notes, keeping credentials out of memory while still recording the operational fix. After the change, I validated the result at both the HTTP layer and in the browser to confirm the site was genuinely public rather than only appearing fixed in API state.

## Outcomes
`brownbio.tech` now returns HTTP 200 instead of 401. Browser verification showed the public Brown Biotech homepage loading successfully, confirming the Vercel protection issue was resolved.
